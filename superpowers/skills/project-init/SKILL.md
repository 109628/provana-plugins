# project-init — New Project Bootstrap

> Invoke when: starting a brand-new Provana project from zero (git init, Azure RG, context wiring).
> One command to production-ready structure. Covers all three pod types.

---

## When to invoke

- "Start a new project"
- "Bootstrap [project name]"
- "Create the repo for [feature/pod]"
- `provana-superpowers:project-init`

---

## Overview

A solo developer should go from zero to first commit in under 15 minutes. This skill produces:

1. Git repo with branch structure
2. Azure Resource Group + `.env.azure.[env]`
3. All context files (`CLAUDE.md`, `docs/`, `hooks/`)
4. provana-bootstrap invocation to register the session
5. First commit + first worktree ready for feature work

## Step 2 — Local repo

```bash
# 2a. Create repo
mkdir "$PROJECT" && cd "$PROJECT"
git init
git checkout -b main

# 2b. Copy Provana context templates
cp "$PLUGIN_PATH/context-templates/CLAUDE.md" ./CLAUDE.md
# Replace all placeholders
sed -i "s/REPLACE_PROJECT_NAME/$PROJECT/g" CLAUDE.md
sed -i "s/REPLACE_POD_TYPE/$POD/g" CLAUDE.md

# 2c. Standard directory structure
mkdir -p docs tests/{unit,integration,smoke,qa} src reviews reports .worktrees

# 2d. Pod-specific source dir
case "$POD" in
  conv_ai) mkdir -p src/conv_ai ;;
  doc_ai)  mkdir -p src/doc_ai ;;
  bpm)     mkdir -p src/bpm ;;
esac

# 2e. Initial docs
touch docs/PRD.md docs/arch.md docs/decisions.md

# 2f. Add standard gitignore
cat > .gitignore << 'EOF'
.env*
!.env.example
__pycache__/
*.pyc
.pytest_cache/
.playwright/
node_modules/
dist/
build/
*.egg-info/
.venv/
venv/
.worktrees/
/tmp/provana-*
EOF

# 2g. Copy hooks
cp -r "$PLUGIN_PATH/hooks" ./hooks
chmod +x hooks/*.sh

# 2h. requirements stubs
cat > requirements.txt << 'EOF'
# Runtime dependencies — fill in per pod
EOF

cat > requirements-test.txt << 'EOF'
pytest>=8.0
pytest-asyncio>=0.23
pytest-cov>=5.0
pytest-xdist>=3.5
pytest-rerunfailures>=14.0
EOF
```

## Step 3 — Branch structure

```bash
# Provana standard branch model
git add -A
git commit -m "chore(init): bootstrap $PROJECT — Provana AI-Native Delivery v2.0"

# Create integration branch off main
git checkout -b develop
git checkout main

# Working branches follow: feat/[pod]-[story] | fix/[pod]-[issue] | infra/[component]
echo "Repo ready. Working branches: feat/$POD-* | fix/$POD-* | infra/*"
```

## Step 4 — Azure foundation

```bash
# 4a. Login check
az account show --query "{sub:id, name:name}" -o table
az account set --subscription "$AZURE_SUBSCRIPTION"

# 4b. Resource group
RG_NAME="rg-$PROJECT-$AZURE_ENV"
az group create \
  --name "$RG_NAME" \
  --location "$AZURE_REGION" \
  --tags project="$PROJECT" environment="$AZURE_ENV" owner="$ALERT_EMAIL"

# 4c. Seed .env file (appended to by azure-deployment steps)
cat > ".env.azure.$AZURE_ENV" << EOF
# Auto-generated by project-init — $(date -u '+%Y-%m-%dT%H:%M:%SZ')
# DO NOT COMMIT — listed in .gitignore
PROJECT=$PROJECT
AZURE_ENV=$AZURE_ENV
AZURE_REGION=$AZURE_REGION
AZURE_SUBSCRIPTION=$AZURE_SUBSCRIPTION
RESOURCE_GROUP=$RG_NAME
ALERT_EMAIL=$ALERT_EMAIL
EOF

echo "Resource group $RG_NAME created in $AZURE_REGION"
```

## Quick-start cheat sheet

```bash
# Minimum viable bootstrap (copy-paste, fill variables at top)
PROJECT=my-project
POD=doc_ai
AZURE_ENV=dev
AZURE_REGION=eastus2
AZURE_SUBSCRIPTION=$(az account show --query id -o tsv)
ALERT_EMAIL=you@provana.com
PLUGIN_PATH=~/.claude/plugins/provana-superpowers

mkdir $PROJECT && cd $PROJECT && git init && git checkout -b main
cp $PLUGIN_PATH/context-templates/CLAUDE.md ./CLAUDE.md
sed -i "s/REPLACE_PROJECT_NAME/$PROJECT/g; s/REPLACE_POD_TYPE/$POD/g" CLAUDE.md
mkdir -p docs tests/{unit,integration,smoke,qa} src/$POD reviews reports .worktrees
cp -r $PLUGIN_PATH/hooks ./hooks && chmod +x hooks/*.sh
touch docs/PRD.md docs/arch.md docs/decisions.md requirements.txt requirements-test.txt
git add -A && git commit -m "chore(init): bootstrap $PROJECT"
az group create --name rg-$PROJECT-$AZURE_ENV --location $AZURE_REGION \
  --tags project=$PROJECT environment=$AZURE_ENV owner=$ALERT_EMAIL
echo "Done. Next: provana-superpowers:bmad-discovery"
```

## Red flags

- Project name >24 chars → Azure resource names will fail. Shorten first.
- `.env.azure.*` in git → blocked by pre-commit.sh secrets scan. Good — do not override.
- Skipping `docs/PRD.md` → bmad-discovery will create one anyway, but starting with a stub saves a round-trip.
- Running `project-init` on an existing repo → non-destructive (cp does not overwrite existing CLAUDE.md), but double-check with `git diff` before committing.
