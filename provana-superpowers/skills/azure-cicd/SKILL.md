# azure-cicd — Azure DevOps Pipeline Generator

> Invoke when: wiring automated build → test → deploy for any Provana project on Azure DevOps.
> Generates complete azure-pipelines.yml with Provana quality gates integrated.

---

## When to invoke

- "Set up CI/CD"
- "Create the Azure DevOps pipeline"
- "Automate deployments"
- "Wire up the pipeline for [project]"
- `provana-superpowers:azure-cicd`

---

## Overview

This skill generates a production-grade `azure-pipelines.yml` that:

1. **Runs on every PR** — build, lint, unit tests, quality gate (`post-test.sh`)
2. **Runs on merge to develop** — integration tests, deploy to `dev`
3. **Runs on merge to main** — full test suite, deploy to `staging` then `prod` (with approval gate)
4. Integrates the Provana `post-test.sh` merge gate as a pipeline task
5. Publishes test results and coverage to Azure DevOps dashboards

---

## Step 1 — Prerequisites

```bash
# Required before generating pipeline:
# 1. Azure DevOps project exists
# 2. Service connection created (ARM service principal)
# 3. Azure Container Registry exists (from azure-deployment step 02)
# 4. Variable group created in Azure DevOps Library

# Create variable group (one-time setup)
az pipelines variable-group create \
  --name "provana-$PROJECT-$AZURE_ENV" \
  --variables \
    PROJECT="$PROJECT" \
    AZURE_ENV="$AZURE_ENV" \
    ACR_NAME="${PROJECT}${AZURE_ENV}acr" \
    RESOURCE_GROUP="rg-$PROJECT-$AZURE_ENV" \
    ALERT_EMAIL="$ALERT_EMAIL" \
  --authorize true
```

## Step 2 — Generate azure-pipelines.yml

Replace `REPLACE_*` tokens before committing:

```yaml
# azure-pipelines.yml
# Provana AI-Native Delivery — CI/CD Pipeline
# Generated by provana-superpowers:azure-cicd

trigger:
  branches:
    include:
      - main
      - develop
  paths:
    exclude:
      - docs/**
      - '*.md'

pr:
  branches:
    include:
      - main
      - develop

variables:
  - group: provana-REPLACE_PROJECT-dev   # Variable group from Step 1
  - name: pythonVersion
    value: '3.11'
  - name: imageRepository
    value: '$(PROJECT)-api'
  - name: dockerfilePath
    value: '$(Build.SourcesDirectory)/Dockerfile'
  - name: tag
    value: '$(Build.BuildId)'

stages:

# ─────────────────────────────────────────────
# STAGE 1: Quality Gate (every PR + push)
# ─────────────────────────────────────────────
- stage: QualityGate
  displayName: 'Quality Gate'
  jobs:
  - job: Lint
    displayName: 'Lint & Static Analysis'
    pool:
      vmImage: 'ubuntu-22.04'
    steps:
    - task: UsePythonVersion@0
      inputs:
        versionSpec: '$(pythonVersion)'

    - script: |
        pip install ruff mypy --break-system-packages 2>/dev/null || pip install ruff mypy
        ruff check src/ --output-format=github
        mypy src/ --ignore-missing-imports
      displayName: 'Ruff + mypy'

  - job: UnitTests
    displayName: 'Unit Tests + Provana Gate'
    pool:
      vmImage: 'ubuntu-22.04'
    steps:
    - task: UsePythonVersion@0
      inputs:
        versionSpec: '$(pythonVersion)'

    - script: |
        pip install -r requirements.txt -r requirements-test.txt \
          --break-system-packages 2>/dev/null || \
          pip install -r requirements.txt -r requirements-test.txt
      displayName: 'Install dependencies'

    - script: |
        pytest tests/unit/ \
          --tb=short \
          --cov=src \
          --cov-report=xml:coverage.xml \
          --cov-report=term-missing \
          --junitxml=test-results.xml \
          -n auto
      displayName: 'pytest unit'

    - task: PublishTestResults@2
      condition: always()
      inputs:
        testResultsFormat: 'JUnit'
        testResultsFiles: 'test-results.xml'
        failTaskOnFailedTests: true

    - task: PublishCodeCoverageResults@2
      condition: always()
      inputs:
        summaryFileLocation: 'coverage.xml'
        failIfCoverageEmpty: false

    - script: |
        # Provana post-test.sh quality gate
        chmod +x hooks/post-test.sh
        bash hooks/post-test.sh
      displayName: 'Provana merge gate'
      env:
        PYTEST_EXIT_CODE: $(pytest_exit_code)

# ─────────────────────────────────────────────
# STAGE 2: Build & Push (develop + main)
# ─────────────────────────────────────────────
- stage: Build
  displayName: 'Build & Push Image'
  dependsOn: QualityGate
  condition: and(succeeded(), in(variables['Build.SourceBranch'], 'refs/heads/develop', 'refs/heads/main'))
  jobs:
  - job: BuildImage
    displayName: 'Docker Build → ACR'
    pool:
      vmImage: 'ubuntu-22.04'
    steps:
    - task: AzureCLI@2
      displayName: 'ACR login'
      inputs:
        azureSubscription: 'REPLACE_SERVICE_CONNECTION'
        scriptType: 'bash'
        scriptLocation: 'inlineScript'
        inlineScript: |
          az acr login --name $(ACR_NAME)

    - task: Docker@2
      displayName: 'Build and push'
      inputs:
        command: 'buildAndPush'
        containerRegistry: '$(ACR_NAME)'
        repository: '$(imageRepository)'
        dockerfile: '$(dockerfilePath)'
        tags: |
          $(tag)
          latest

# ─────────────────────────────────────────────
# STAGE 3: Deploy Dev (develop branch only)
# ─────────────────────────────────────────────
- stage: DeployDev
  displayName: 'Deploy → Dev'
  dependsOn: Build
  condition: and(succeeded(), eq(variables['Build.SourceBranch'], 'refs/heads/develop'))
  variables:
    - group: provana-REPLACE_PROJECT-dev
  jobs:
  - deployment: DeployContainerAppDev
    displayName: 'Container App — dev'
    environment: 'dev'
    pool:
      vmImage: 'ubuntu-22.04'
    strategy:
      runOnce:
        deploy:
          steps:
          - task: AzureCLI@2
            displayName: 'Update Container App image'
            inputs:
              azureSubscription: 'REPLACE_SERVICE_CONNECTION'
              scriptType: 'bash'
              scriptLocation: 'inlineScript'
              inlineScript: |
                az containerapp update \
                  --name "$(PROJECT)-dev-ca" \
                  --resource-group "$(RESOURCE_GROUP)" \
                  --image "$(ACR_NAME).azurecr.io/$(imageRepository):$(tag)"

          - task: AzureCLI@2
            displayName: 'Smoke tests'
            inputs:
              azureSubscription: 'REPLACE_SERVICE_CONNECTION'
              scriptType: 'bash'
              scriptLocation: 'inlineScript'
              inlineScript: |
                pip install pytest httpx --break-system-packages 2>/dev/null || \
                  pip install pytest httpx
                pytest tests/smoke/ --tb=short -q

# ─────────────────────────────────────────────
# STAGE 4: Deploy Staging + Prod (main branch)
# ─────────────────────────────────────────────
- stage: DeployStaging
  displayName: 'Deploy → Staging'
  dependsOn: Build
  condition: and(succeeded(), eq(variables['Build.SourceBranch'], 'refs/heads/main'))
  variables:
    - group: provana-REPLACE_PROJECT-staging
  jobs:
  - deployment: DeployStaging
    displayName: 'Container App — staging'
    environment: 'staging'   # approval gate configured in Azure DevOps UI
    pool:
      vmImage: 'ubuntu-22.04'
    strategy:
      runOnce:
        deploy:
          steps:
          - task: AzureCLI@2
            displayName: 'Update staging Container App'
            inputs:
              azureSubscription: 'REPLACE_SERVICE_CONNECTION'
              scriptType: 'bash'
              scriptLocation: 'inlineScript'
              inlineScript: |
                az containerapp update \
                  --name "$(PROJECT)-staging-ca" \
                  --resource-group "rg-$(PROJECT)-staging" \
                  --image "$(ACR_NAME).azurecr.io/$(imageRepository):$(tag)"

          - script: |
              pip install pytest httpx --break-system-packages 2>/dev/null || \
                pip install pytest httpx
              pytest tests/smoke/ tests/integration/ --tb=short -q
            displayName: 'Integration + smoke'

- stage: DeployProd
  displayName: 'Deploy → Production'
  dependsOn: DeployStaging
  condition: succeeded()
  variables:
    - group: provana-REPLACE_PROJECT-prod
  jobs:
  - deployment: DeployProd
    displayName: 'Container App — prod'
    environment: 'prod'    # REQUIRES manual approval in Azure DevOps UI
    pool:
      vmImage: 'ubuntu-22.04'
    strategy:
      runOnce:
        deploy:
          steps:
          - task: AzureCLI@2
            displayName: 'Blue/green swap — prod'
            inputs:
              azureSubscription: 'REPLACE_SERVICE_CONNECTION'
              scriptType: 'bash'
              scriptLocation: 'inlineScript'
              inlineScript: |
                # Update with traffic revision
                az containerapp update \
                  --name "$(PROJECT)-prod-ca" \
                  --resource-group "rg-$(PROJECT)-prod" \
                  --image "$(ACR_NAME).azurecr.io/$(imageRepository):$(tag)" \
                  --revision-suffix "build-$(tag)"

                # Wait for new revision to be provisioned
                az containerapp revision list \
                  --name "$(PROJECT)-prod-ca" \
                  --resource-group "rg-$(PROJECT)-prod" \
                  --query "[0].properties.runningState" -o tsv

          - script: |
              pip install pytest httpx --break-system-packages 2>/dev/null || \
                pip install pytest httpx
              pytest tests/smoke/ -k "prod" --tb=short -q
            displayName: 'Production smoke test'
```

## Step 3 — Function App pipeline variant

When the project uses Azure Functions instead of Container Apps, replace the deploy steps:

```yaml
# Replace DeployDev deploy steps with:
- task: AzureFunctionApp@2
  displayName: 'Deploy Function App'
  inputs:
    azureSubscription: 'REPLACE_SERVICE_CONNECTION'
    appType: 'functionAppLinux'
    appName: '$(PROJECT)-$(AZURE_ENV)-func'
    package: '$(Build.ArtifactStagingDirectory)/**/*.zip'
    runtimeStack: 'PYTHON|3.11'
```

Build step to add before deploy for Functions:

```yaml
- script: |
    pip install -r requirements.txt --target=".python_packages/lib/site-packages"
    cd $(Build.SourcesDirectory)
    zip -r $(Build.ArtifactStagingDirectory)/function.zip . \
      -x "*.git*" "*test*" "*.venv*" "*__pycache__*"
  displayName: 'Package function app'

- task: PublishBuildArtifacts@1
  inputs:
    pathToPublish: '$(Build.ArtifactStagingDirectory)'
    artifactName: 'function-package'
```

## Step 4 — Environment approval gates (one-time Azure DevOps setup)

```
Azure DevOps → Pipelines → Environments → [env name] → Approvals and checks
→ Add "Approvals" check
→ Approvers: [tech lead email]
→ Timeout: 4 hours
→ Instructions: "Verify staging smoke tests pass before approving prod"
```

Configure for `staging` (optional, recommended) and `prod` (mandatory).

## Step 5 — Service connection setup

```bash
# Create service principal for pipeline
SP_NAME="sp-$PROJECT-cicd"
SP_JSON=$(az ad sp create-for-rbac \
  --name "$SP_NAME" \
  --role Contributor \
  --scopes "/subscriptions/$AZURE_SUBSCRIPTION/resourceGroups/rg-$PROJECT-dev" \
             "/subscriptions/$AZURE_SUBSCRIPTION/resourceGroups/rg-$PROJECT-staging" \
             "/subscriptions/$AZURE_SUBSCRIPTION/resourceGroups/rg-$PROJECT-prod" \
  --sdk-auth)

echo "Add to Azure DevOps → Project Settings → Service connections → New → Azure Resource Manager"
echo "Use 'Service principal (manual)' and paste the JSON above"
echo ""
echo "Service connection name: REPLACE_SERVICE_CONNECTION → use: $SP_NAME"
```

After generating the file:

## Rollback procedure

When a production deployment fails or post-deploy smoke tests fail:

```bash
# List recent revisions
az containerapp revision list \
  --name "$PROJECT-prod-ca" \
  --resource-group "rg-$PROJECT-prod" \
  --query "[].{name:name, active:properties.active, created:properties.createdTime}" \
  --output table

# Activate previous revision (replace REVISION_NAME)
az containerapp revision activate \
  --name "$PROJECT-prod-ca" \
  --resource-group "rg-$PROJECT-prod" \
  --revision PREVIOUS_REVISION_NAME

# Deactivate failed revision
az containerapp revision deactivate \
  --name "$PROJECT-prod-ca" \
  --resource-group "rg-$PROJECT-prod" \
  --revision FAILED_REVISION_NAME
```

For Function Apps:

```bash
# Swap back to previous deployment slot
az functionapp deployment slot swap \
  --name "$PROJECT-$AZURE_ENV-func" \
  --resource-group "rg-$PROJECT-$AZURE_ENV" \
  --slot staging \
  --target-slot production
```

**Record rollback in decisions.md:**

```markdown
| RB-001 | YYYY-MM-DD | Rolled back prod to build-[N-1] | Resolved | Build-N | Smoke test failure on /api/health |
```

## Red flags

- Skipping environment approval gates on prod → never acceptable for production workloads.
- Hardcoding secrets in azure-pipelines.yml → use Variable Groups with Azure Key Vault linkage instead.
- Deploying on red quality gate → `post-test.sh` exits 1, pipeline stops. Fix tests before merging.
- Single variable group for all envs → each env must have its own variable group with env-scoped values.
